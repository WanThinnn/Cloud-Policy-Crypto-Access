Access Policy Management (ABAC Layer)

Detailed conditions for permissions - Applied AFTER basic permissions from User Types

📋 How Hybrid RBAC + ABAC works

Layer 1: RBAC (User Types)

Basic permissions from role. Ex: Data User can file_read, file_download

→ If NO basic permissions → DENIED
Layer 2: ABAC (These Policies)

Detailed conditions. Ex: Only IT department, secret+ security level can read technical documents

→ Fine-tune permissions according to specific context
Note: DENY policy always takes precedence over ALLOW. No policy -> use RBAC result.
